# SOP — Як користуватися skill `clipping`

Цей документ — покрокова інструкція для людини, яка хоче нарізати довге відео на короткі кліпи через цей skill.

---

## Що це робить

Беремо одне довге відео + транскрипт із таймкодами → отримуємо набір окремих кліпів, по одному на кожен сегмент із транскрипту. Без склейок, без хуків, без A/B варіантів.

---

## Що потрібно перед стартом

### 1. Встановити ffmpeg

**macOS:**
```bash
brew install ffmpeg
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt update && sudo apt install ffmpeg
```

**Windows:**
Завантажити з https://ffmpeg.org/download.html і додати до PATH.

Перевірити що працює:
```bash
ffmpeg -version
```

### 2. Підготувати файли

Тобі потрібні **два** файли в одній папці:

1. **Відео** — `.mov` або `.mp4` (наприклад `video.mov`)
2. **Транскрипт** з таймкодами — `.txt`, `.md` або `.docx` із позначеними початком/кінцем кожного сегмента

**Приклад транскрипту:**
```
[00:01:05 — 00:02:30] Тема 1: Як починати бізнес
"Текст того що говорить спікер..."

[00:05:12 — 00:07:45] Тема 2: Помилки новачків
"Текст..."

[00:10:00 — 00:12:30] Тема 3: Перші клієнти
"Текст..."
```

Головне щоб були видні **час початку**, **час кінця** і **назва теми** для кожного сегмента.

## Як запустити skill

### Варіант А — через Claude Code (рекомендую)

1. Відкрий папку з відео і транскриптом у Claude Code
2. Переконайся, що skill `clipping` встановлений (папка `clipping/` лежить у проєкті або в `~/.claude/skills/clipping/`)
3. Напиши Claude приблизно так:

   > "Наріж відео `video.mov` за транскриптом `transcript.txt`. Використай skill clipping."

4. Claude:
   - Прочитає транскрипт
   - Витягне таймкоди і назви тем
   - Застосує буфери (5с до / 1с після)
   - Нарізає кожен сегмент через ffmpeg із re-encode
   - Збереже в папку `clips/`

5. Перевір результат у папці `clips/`

### Варіант Б — вручну через термінал

Якщо хочеш нарізати один кліп руками:

```bash
bash clipping/scripts/cut_clip.sh \
  video.mov \
  "00:01:05" \
  "00:02:30" \
  "Clip_1_TopicName"
```

Скрипт сам додасть буфери і запустить ffmpeg із правильними параметрами.

## Що отримаєш на виході

У папці `clips/` зʼявляться файли:

```
clips/

├── Clip_2_ПомилкиНовачків.mp4
├── Clip_3_ПершіКлієнти.mp4

```

Кожен файл — окремий кліп, готовий до публікації.

## Як перевірити що все добре

Для кожного кліпа перевір:

- [ ] **Перші слова не зʼїдені** — спікер починає говорити не з першої секунди
- [ ] **Кінець не різко обірваний** — є секунда тиші в кінці
- [ ] **Звук і відео синхронні** — губи відповідають голосу
- [ ] **Назва файлу зрозуміла** — видно про що кліп

Якщо щось не так:
- Перші слова зʼїдені → попроси Claude збільшити буфер до **7-8 секунд** до початку
- Різкий кінець → попроси збільшити буфер після до **2 секунд**
- Звук рассинхрон → це значить що використалось `-c copy`. **Переробити з re-encode** (skill це робить за замовчуванням, але переконайся)

---

## Чого НЕ робити

- **НЕ використовувати `-c copy`** — це швидко, але зʼїдає перші слова і ламає звук
- **НЕ ставити буфери менші ніж 5с/1с** — без буфера часто обрізає початок
- **НЕ покладатися на точність таймкодів у транскрипті** — вони приблизні, краще перевіряти на слух

---

## Часті проблеми

| Проблема | Причина | Рішення |
|---|---|---|
| Перші слова зʼїдені | Буфер замалий або `-c copy` | Збільшити буфер до 7-8с, re-encode |
| Звук відстає від відео | Використано `-c copy` | Пере-нарізати з re-encode |
| Файл не зберігається | Папки `clips/` немає | Створити `mkdir clips` перед запуском |
| `ffmpeg: command not found` | ffmpeg не встановлений | Встановити (див. початок документа) |
| Кліп задовгий | Таймкоди в транскрипті неточні | Підправити end timecode вручну |
